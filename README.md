# 出来高表（dekidaka）

標識施工会社向けの **日次出来高・工事原価・利益管理** デスクトップアプリ。
合同会社（複数メンバー社）が **1台のPCで集中管理** して使う前提のフル機能版。

- フレームワーク：Vite + React + TypeScript + Zustand
- デスクトップ化：**Tauri 2**（macOS `.dmg` / Windows `.msi`・`.exe`）
- データ保存：**2モード自動切替**
  - **サーバーモード**（推奨・複数PC共有）：**Supabase**（無料）に全データ保存。`VITE_SUPABASE_URL` / `VITE_SUPABASE_ANON_KEY` を設定すると有効。
  - **ローカルモード**（開発・1台運用）：未設定時はこのPC内の JSON に保存。
- グラフ：Recharts

## 複数PCで使う（サーバーモード）

1. **`docs/サーバー設定手順.md`** に沿って Supabase プロジェクトを作成し、`supabase/schema.sql` を実行。
2. 接続情報を `.env`（開発時）または GitHub Secrets（配布ビルド時）に設定：
   ```
   VITE_SUPABASE_URL=...
   VITE_SUPABASE_ANON_KEY=...
   ```
3. 初回起動時の「はじめての設定」で最初の管理者を作成。以降はログインして全PCで同じデータを共有。
4. 各PCへの導入は **`docs/インストール手順.md`** を参照。

## 主な機能

| 区分 | 内容 |
|------|------|
| ダッシュボード | 当月/累計KPI、月次推移、予実比較、会社別利益、進行中工事の進捗 |
| 出来高 入力 | 単価マスター選択＋本数で売上自動計算、車両チェックで原価加算、**総利益ライブ表示**、下書き→申請 |
| 出来高表 一覧 | 期間/会社/工事/班/状態フィルタ、承認、複製、削除、合計行 |
| 工事台帳 | 工番/契約金額/工期、契約 vs 累積出来高（進捗率）、工事別損益 |
| 帳票・印刷 | **Excel風の出来高報告書（A4・ロゴ＋会社名）** をPDF印刷、月次/会社別/工事別集計、CSV出力 |
| マスター設定 | 会社（ロゴ）/施工班/車両（日額）/単価/発注元/予算/ユーザー、1人あたり日当 |
| 管理 | 監査ログ、バックアップ/リストア、デモ投入、全初期化 |
| 権限 | 管理者 / 経理 / 現場責任者 / 閲覧 のロール別制御 |

## 計算式

```
人件費 = 人数 × 1人あたり日当（設定値）
車両費 = Σ(使用車両の日額)
出来高 = Σ(明細: 単価 × 数量)        ← 売上
原価   = 人件費 + 高速代 + 燃料代 + 車両費
総利益 = 出来高 − 原価
利益率 = 総利益 / 出来高 × 100
```

## 開発

```bash
npm install
npm run dev          # ブラウザ開発（http://localhost:5173）
npm run tauri dev    # ネイティブ（Tauri）で起動
npm test             # 計算ロジックの単体テスト（Vitest）
```

## ビルド（インストーラ生成）

```bash
npm run tauri build  # 実行OS向けにビルド（Mac=.dmg / Windows=.msi）
```

- macOS と Windows の両方を配布する場合は GitHub Actions（`.github/workflows/release.yml`）を使用。
  `vX.Y.Z` タグを push すると両OSのインストーラがドラフトReleaseに添付されます。

## 初期アカウント

- **サーバーモード**：初回起動の「はじめての設定」画面で最初の管理者を作成します。以降のユーザーは
  「設定 → ユーザー」で管理者が発行します。
- **ローカルモード**（開発時）：デモデータに以下が含まれます。

| ログインID | パスワード | 権限 |
|-----------|-----------|------|
| `admin` | `admin` | 管理者 |
| `keiri` | `1234` | 経理 |
| `genba` | `1234` | 現場責任者 |

> ローカルモードは初回にデモデータが入ります。実運用前に「管理 → 全データ初期化」で空にし、
> マスター（会社・単価・車両・日当など）を登録してください。

## データの保存場所

- アプリのデータディレクトリ内 `dekidaka-data.json`（ブラウザ開発時は localStorage）。
- 「管理 → バックアップを保存」で任意の場所へ書き出し、「復元」で読み込めます。
