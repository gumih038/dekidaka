# サーバー設定手順（管理者向け・最初に1回だけ）

複数のPCで同じデータを共有するため、無料サーバー **Supabase** を使います。
この作業は**最初に1回だけ**、詳しい人（管理者）が行えばOKです。所要 約15分・無料。

---

## ステップ1：Supabase でプロジェクトを作る

1. ブラウザで <https://supabase.com> を開き、「Start your project」からサインアップ（GitHubアカウントでログインが簡単）。
2. 「New project」を押す。
   - **Name**：`dekidaka`（何でもOK）
   - **Database Password**：自動生成でOK（控えなくて大丈夫）
   - **Region**：`Northeast Asia (Tokyo)` を選ぶ
3. 「Create new project」を押し、1〜2分待つ（サーバー準備中）。

## ステップ2：データの入れ物を作る（SQLを貼るだけ）

1. 左メニューの **SQL Editor** を開く →「New query」。
2. このリポジトリの **`supabase/schema.sql`** の中身を全部コピーして貼り付ける。
3. 右下の **Run** を押す。「Success」と出ればOK。

## ステップ3：メール確認をオフにする（重要）

社内利用なのでメール確認なしで使えるようにします。

1. 左メニュー **Authentication** → **Sign In / Providers**（または Providers）→ **Email** を開く。
2. **Confirm email**（メール確認）を **OFF** にして保存。

## ステップ4：接続情報（URLとキー）を控える

1. 左メニュー **Project Settings**（歯車）→ **API**。
2. 次の2つをメモ：
   - **Project URL**（例：`https://abcd1234.supabase.co`）
   - **anon public** キー（`eyJ...` で始まる長い文字列）

> anon キーはアプリに埋め込みます。公開されても、ログインした人だけがデータを読み書きできる設定（RLS）になっているので安全です。

---

## ステップ5：GitHub でインストーラを作る準備

Windows用インストーラは GitHub の無料サービスで自動作成します。

1. <https://github.com> でアカウントを作成（無ければ）。
2. 新しいリポジトリ（例：`dekidaka`、Private推奨）を作り、このプロジェクト一式を push する。
   ```bash
   cd dekidaka
   git init
   git add .
   git commit -m "initial"
   git branch -M main
   git remote add origin https://github.com/＜あなた＞/dekidaka.git
   git push -u origin main
   ```
3. リポジトリの **Settings → Secrets and variables → Actions → New repository secret** で、
   ステップ4の値を2つ登録：
   | Name | Secret |
   |------|--------|
   | `VITE_SUPABASE_URL` | ステップ4の Project URL |
   | `VITE_SUPABASE_ANON_KEY` | ステップ4の anon public キー |

## ステップ6：インストーラを作る（タグを付けるだけ）

```bash
git tag v0.2.0
git push origin v0.2.0
```

- GitHub の **Actions** タブで自動ビルドが走ります（約10分）。
- 終わると **Releases**（下書き）に Windows用 `.msi` / `.exe` と Mac用 `.dmg` が添付されます。
- Release を「Publish」し、`.msi` をダウンロード → 社内ドライブ（Google Drive等）に置く。

あとは各PCで `.msi` を入れるだけです（→ `インストール手順.md`）。

---

## 困ったとき
- ログインできない／管理者が作れない → ステップ3「Confirm email を OFF」を確認。
- データが他のPCに出ない → 各PCが同じ `VITE_SUPABASE_URL` のインストーラを使っているか確認。
- 最初の管理者は、アプリ初回起動時の「はじめての設定」画面で作成します。
